Scores every eligible action and returns the best one.

Skips null and ineligible (ITactixAction::IsValid is false) actions. Strictly-greater comparison means ties resolve to the lower index, which keeps the choice stable from frame to frame when scores are equal.

Parameters
ActionsArray of candidate actions; entries may be null.
CountNumber of entries in Actions.
CtxRead-only agent state passed to each action.
Returns
The winner, or a result with FTactixUtilityResult::HasWinner false if nothing scored above zero.

Combines a set of considerations into one score in [0, 1].

A plain product of considerations is biased: every extra factor below 1 drags the result down, so an action with more considerations loses to one with fewer for no good reason. This applies the Mike Lewis make-up factor (from the IAUS literature) to counteract that bias, scaling each term back up based on how many considerations there are.

Parameters
ConsiderationsArray of considerations; a null entry forces 0.
CountNumber of considerations.
CtxRead-only agent state.
Returns
The compensated product in [0, 1]. Returns 0 immediately if any consideration evaluates to 0 (a hard veto) or the input is empty.
