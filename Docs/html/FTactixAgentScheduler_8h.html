Detailed Description

Min-heap tick distributor that spreads agent work across frames under a per-frame CPU budget.

This exists to answer "how do I tick 200 agents in a 10 ms frame?". Ticking every agent every frame doesn't fit, so each agent instead registers how often it wants to think (TickIntervalMs) and roughly how long a think costs (BudgetUs). Once a frame the game thread calls Collect, which drains the agents that are due, in time order, and stops as soon as the frame's CPU budget is used up. Agents that didn't fit stay at the head of the heap and get first pick next frame, so nobody starves indefinitely.

Implementation choices worth knowing:

  • The heap is hand-rolled over a heap-allocated array; std::vector and std::priority_queue are off-limits per the project rules.
  • Entries are stored by value, not by pointer. Each is small, so a few hundred agents stay within a couple of cache-friendly kilobytes.
  • ReportTickCost is optional but recommended: feed back real timings and the next Collect uses them instead of the estimate.
