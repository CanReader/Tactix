Detailed Description

template<std::size_t Capacity = 64>
class Tactix::FTactixBlackboard< Capacity >

Concrete fixed-capacity blackboard backed by an open-addressed table.

Linear probing with backshift deletion, no heap allocation, no UObject. The table never resizes; pick Capacity comfortably above the number of keys you expect, since a full table silently drops new inserts.

Template Parameters
CapacitySlot count. Must be a power of two so the probe wrap is a bitmask. Default 64.

template<std::size_t Capacity = 64>
bool Tactix::FTactixBlackboard< Capacity >::Remove ( FTactixBBKey Key)
inlineoverridevirtual

Removes a key and repairs the probe chain behind it.

After clearing the slot, this does a Robin-Hood-style backshift: it walks forward and drags back any entry that probed past the now-empty slot, so later Find calls don't terminate early at the hole and miss a key.
